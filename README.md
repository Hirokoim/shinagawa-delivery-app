- 4つのコア画面（入退去申請、工事申請、AI検証、ルールマスタ管理）

- 条件分岐するフォーム

## 開発ワークフロー

Claude Code を使った構造化ワークフロー：

1. Explore — 関連ドキュメントとソースを読み、現状を整理
2. Plan — 実装プランを提案し、承認を待つ
3. Code — 承認されたプランのみを小さく実装
4. Review — 変更点とテスト内容をまとめる
5. Commit — Conventional Commits 形式

詳細は docs/11_development_workflow.md を参照。
